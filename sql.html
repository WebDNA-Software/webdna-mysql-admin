[!] sql.html[?db=DB&q=QUERY] — run arbitrary SQL. Result metadata ([numfound]/[numrowsaffected]/[insertid]) always works. GRID RENDERING for SELECT results is the hard part: WebDNA can read a result column by NAME but has no documented way to *enumerate* an arbitrary result's column names. So for a clean single SELECT/WITH with a database selected we MATERIALIZE it into a scratch table, read its columns from information_schema, render the grid with the dynamic [interpret][[col]][/interpret] trick, then drop it. Requires CREATE/DROP on the selected database. Non-SELECT statements (INSERT/UPDATE/DDL/...) just execute and report rows affected. Grid mode is used only when ALL of these hold (else: metadata-only): * a database is selected * the query begins with SELECT or WITH * the query has no ';' (single statement; don't terminate with a semicolon) WebDNA-processed.
